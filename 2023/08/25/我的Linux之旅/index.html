我的Linux之旅

从一个windows小白,到Arch Linux

Tips: 假如涉及到你不太了解的名词，请自行Google

Windows

大一时，我还是个windows小白。家里没有电脑，也没有去过网吧，所以一开始还和垃圾软件作过一段时间的斗争。当时不知道自由(开源)软件，也不知道Github，更不知道windows居然有包管，只知道打开浏览器下载软件，还用的是百毒搜索引擎，前几个必是xxx软件园。否则，我现在估计也还是会用windows的。从这个角度来看的话，还真的是要感谢百度了。

Linux初体验

第一次体验GNU/Linux distro，并没有觉得有什么了不起，因为属于我的蒙娜丽莎，我早已遇见。

我第一眼见到的是比特鹏哥在vmware中装的centOS，他主要是出于教学目的来演示GCC编译器。说实话，centOS确实比较老了，无论哪个DE都很丑。但是学习基础命令的话，还是挺合适的，中文资料也比较丰富。大概很多人都是从centOS或者Ubuntu开始起步的，毕竟对新手特别友好。

现在，对于一个没有接触过Linux的人，我也会推荐他(她)使用Debian(系)。

从centOS到Ubuntu

这个‘从centOS’其实是可以省略的，因为我当时并没有在centOS上停留很多时间，仅仅是装了几次虚拟机。

其实后来还是用过centOS的。Java课设用到了mysql，为了方便他人，我在aliyun上部署了个mysql，centOS上跑的Docker运行mysql。国内的公司还蛮喜欢centOS的，大多数云服务商也都支持centOS。其实，Debian系就已经占据了GNU/Linux distro的半壁江山了，再加上redhat系，几乎就是全部了。BTW, I use Arch.

说是两个distro，其实最大的不同也就包管理器。neofetch的输出不同? 你看，Debian的就像依托…，还是Arch的好看。没有传教的意思，绝对没有。

出轨

Ubuntu的内部错误令人难以忍受。于是，我出轨了。它是那么地令人难以捉摸，每一个男孩子的梦中OS ==> Kali Linux(The quieter you become, the more you can hear)，哪个学计算机的没有做过成为Super Hacker的梦呢？

官方很贴心，直接给了虚拟机的压缩文件。于是，我面临的第一个挑战是安装中文输入法。Kali不会中文，所以想要约它，就必须要有中文输入法这个媒介，或者你英文足够好。没有和centOS深入交流，就不需要输入法了，Ubuntu倒是入乡随俗，学了点中文。

和Kali共度的那段时光真的很愉快。我们聊了很多，关于Github使用ssh认证，教它如何使用China的mirror。最后还送了它一份大礼，Jetbrains家的PyCharm。最终也没能俘获它的芳心，其实我们之间还是有可能的，只不过我没能坚持，出于周围环境的原因。

走上正轨

已经忘记了是从哪个时间段开始沉迷于配置vim了。无意之间，看到了一位大佬(TheCW)，虽然他现在已经在用麦金塔了，我真正意义上开始折腾大概就是从这里开始。

打开他的视频，映入眼帘的是一个‘m’(指neofetch的输出，想必有人已经知道哪个distro了)，我首先想到了小米(PS:对小米印象不错，虽然阴差阳错没用过它家的手机，相较于蓝绿两厂好多了。我是真想给手机刷一个Magisk)，后来才知道，这跟小米没有半毛钱关系。是基于Arch Linux的distro，其名为manjaro。

可以说在manjaro上我做了自己想做的所有事情，也是从manjaro开始装了双系统，体验到了装OS的快乐。可以说从按下开机键的那一刻起，你之后看到的所有东西都是可定制的(windows我只是把task bar移到了顶部，Linux你甚至可以手写或者抄别人写好的CSS样式从0开始定制bar)。

如果说只是能够定制美化的话，那Linux不就成花瓶了？

Linux不仅适合开发者和计算机专业的学生，普通人也可以。实际上，甚至部分维护者也并不是计算机专业的。

以Arch Linux为例，假设你已经科学上网:

  • 开发环境
    • vscode: sudo pacman -S visual-studio-code-bin
    • idea: yay -S intellij-idea-ultimate-edition-jre intellij-idea-ultimate-edition
    • jdk: sudo pacman -S jdk17-openjdk
    • docker: sudo pacman -S docker
    • dbeaver: sudo pacman -S dbeaver
    • …
  • 办公日常
    • wps(global): yay -S wps-office
    • typora: yay -S typora-free
    • Okular(文档阅读器): sudo pacman -S okular
    • scrcpy(安卓投屏): yay -S scrcpy
    • telegram: sudo pacman -S telegram-desktop
    • dingtalk: yay -S yay -S dingtalk-bin
    • wemeet: yay -S wemeet-bin
    • QQ: yay -S linuxqq
    • WeChat(wine): yay -S com.qq.weixin.deepin
  • 视频影音
    • 网易云(无社交): yay -S yesplaymusic
    • 落雪音乐: yay -S lx-music-desktop-bin
    • mpv: sudo pacman -S mpv
    • vlc: sudo pacman -S vlc
    • obs: sudo pacman -S obs-studio
  • 游戏娱乐
    • Steam: sudo pacman -S steam
    • Lutris: sudo pacman -S lutris
  • …

Yes, Arch!

双系统也玩了挺长时间的，出于种种原因，最终还是决定舍弃manjaro安装Arch Linux。

安装Arch Linux本就是一个挑战。

第一次安装Arch在虚拟机里试，我竟然一次成功了(因为不用连wifi)。实机安装的时候就踩坑了，安装之后，没有网络，后来才知道要开dhcpcd…

安装完之后，每天开机yay更新系统，再折腾一下Linux下的工具(ranger,neovim,shell-gpt,寻找同类型的在用的软件的更好替代品之类的)，大概有半年了，它仍然健在。由此可见，我还是很保守的。

现在，我仍然在用Arch Linux。
